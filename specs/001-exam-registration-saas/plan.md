# Implementation Plan: Multi-Tenant SaaS Exam Registration System

**Branch**: `001-exam-registration-saas` | **Date**: 2025-11-19 | **Spec**: `specs/001-exam-registration-saas/spec.md`
**Input**: Feature specification from `/specs/001-exam-registration-saas/spec.md`

**Note**: This plan aligns with the ratified constitution (v1.0.0) and drives `/speckit.tasks`.

## Summary

Deliver a full-stack multi-tenant exam registration platform that lets super admins onboard tenants, tenant admins configure exams, candidates apply with document uploads, reviewers process multi-tier approvals, and the system handles payments, ticket generation, seat allocation, and score management. We will extend the existing Spring Boot DDD backend modules (`exam-*`) and the Next.js 14 frontend (`web/`) while preserving schema-per-tenant isolation, RBAC, AES-encrypted PII, automated audit logging, and observability targets defined in the spec and constitution.

## Technical Context

**Language/Version**: Java 21 + Spring Boot 3.4.1 (backend), TypeScript 5.3 + Next.js 14 (frontend)  
**Primary Dependencies**: Spring Security, Spring Data JPA/Hibernate multitenancy, MapStruct, Testcontainers, Flyway, Redis 7, MinIO SDK, OpenFeign, iText7, OpenTelemetry SDK, Next.js App Router, TanStack Query, React Hook Form + Zod, Shadcn/ui, Playwright  
**Storage**: PostgreSQL 15 (public schema + tenant schemas), Redis 7 (caching, queues), MinIO/S3 (documents), Optional CDN for PDFs  
**Testing**: JUnit 5 + Spring Boot Test + Testcontainers, Mockito, Playwright E2E, Cucumber BDD, ESLint + Jest/React Testing Library for UI components  
**Target Platform**: Containerized Linux services for backend, Node 18+/Vercel-like deployment for Next.js frontend, shared CDN + object storage  
**Project Type**: Full-stack monorepo with backend multi-module Maven project and Next.js frontend  
**Performance Goals**: p95 API latency < 500 ms for candidate flows, auto-review response < 2 s, ticket generation < 5 s/item (10k batch < 10 min), seat allocation 50k candidates < 5 min, CSV import 10k rows < 2 min  
**Constraints**: Enforce schema-per-tenant isolation, RBAC on every entry point, AES encryption for PII, idempotent payment callbacks, ≥80% test coverage for new backend code, structured logs + metrics for onboarding, payments, allocation, ticketing, and score publication  
**Scale/Scope**: 100–500 tenants, 100k concurrent candidates, 50k seat assignments per exam cycle, multi-channel notifications, support for WeChat Pay & Alipay
