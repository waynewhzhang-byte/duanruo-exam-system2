# 用户故事

> 端若数智考盟 - 多租户在线招聘考试报名管理平台

---

## 用户故事 1: 租户入驻和考试创建

**作为** 平台超级管理员
**我希望** 能够创建新的租户
**以便** 租户可以在平台上开展招聘考试业务

### 验收标准

1. **场景1**: 超级管理员创建租户
   - **当** 超级管理员填写租户名称 "ABC公司"、租户代码 "abc-corp"
   - **那么** 系统自动创建 PostgreSQL schema `tenant_abc_corp`
   - **并且** 创建 MinIO bucket `tenant-abc-corp-files`
   - **并且** 创建租户管理员账户

2. **场景2**: 租户管理员配置考试
   - **当** 租户管理员登录后台
   - **那么** 可以创建考试，设置报名时间、考试时间
   - **并且** 可以设置是否收费及金额
   - **并且** 考试保存到租户 schema

3. **场景3**: 租户管理员添加岗位
   - **当** 租户管理员为考试添加岗位
   - **那么** 可以设置岗位名称、招聘人数
   - **并且** 可以设置年龄要求、学历要求、性别要求

4. **场景4**: 租户管理员添加科目
   - **当** 租户管理员为岗位添加考试科目
   - **那么** 可以设置科目类型（笔试、面试、实操）
   - **并且** 可以设置及格分数

---

## 用户故事 2: 考生注册和自动审核

**作为** 考生
**我希望** 能够注册并报名考试
**以便** 提交我的应聘资料

### 验收标准

1. **场景1**: 考生浏览考试
   - **当** 考生访问平台
   - **那么** 可以看到所有已发布的考试列表
   - **并且** 可以按租户筛选

2. **场景2**: 考生填写报名信息
   - **当** 考生选择考试和岗位
   - **那么** 需要填写报名表单（姓名、身份证号等）
   - **并且** 需要上传证件照片和资料

3. **场景3**: 考生提交报名
   - **当** 考生提交报名
   - **那么** 系统执行自动审核
   - **如果** 符合全部自动审核条件 → 状态变为 APPROVED
   - **如果** 不符合自动审核条件 → 进入人工审核队列

4. **场景4**: 考生查看报名状态
   - **当** 考生查看 "我的报名"
   - **那么** 可以看到所有报名记录及当前状态

---

## 用户故事 3: 多级人工审核

**作为** 审核员
**我希望** 能够审核考生的报名资料
**以便** 决定是否通过考生的报名

### 验收标准

1. **场景1**: 审核员查看任务队列
   - **当** 审核员登录系统
   - **那么** 可以看到待审核的任务列表
   - **并且** 区分一审和二审队列

2. **场景2**: 审核员拉取任务
   - **当** 审核员点击拉取下一个任务
   - **那么** 系统分配一个任务给该审核员
   - **并且** 任务锁定10分钟防止重复审核

3. **场景3**: 审核员查看报名详情
   - **当** 审核员查看报名
   - **那么** 可以看到报名表全部信息
   - **并且** 可以在线查看上传的证件资料

4. **场景4**: 审核员做出决策
   - **当** 审核员审核报名
   - **那么** 可以选择：通过/拒绝/退回补充材料
   - **并且** 需要填写理由

5. **场景5**: 一审通过后进入二审
   - **当** 一审审核员通过报名
   - **那么** 报名自动进入二审队列
   - **并且** 二审审核员可以看到

6. **场景6**: 二审做出最终决定
   - **当** 二审审核员审核
   - **那么** 通过后状态变为 APPROVED
   - **并且** 考生可以进行后续操作（支付等）

---

## 用户故事 4: 支付和准考证

**作为** 考生
**我希望** 支付报名费并获取准考证
**以便** 能够参加考试

### 验收标准

1. **场景1**: 发起支付
   - **当** 考生报名通过审核且需要缴费
   - **那么** 可以发起支付
   - **并且** 可以选择支付方式（支付宝/微信）

2. **场景2**: 完成支付
   - **当** 考生完成支付
   - **那么** 系统收到支付回调
   - **并且** 更新报名状态为 PAID
   - **并且** 自动生成准考证

3. **场景3**: 查看准考证
   - **当** 考生查看准考证
   - **那么** 可以看到考试名称、考点、时间
   - **并且** 可以下载PDF

4. **场景4**: 座位分配后更新准考证
   - **当** 管理员完成座位分配
   - **那么** 准考证自动更新座位信息
   - **并且** 考生可以看到考场、教室、座位号

---

## 用户故事 5: 考场和座位管理

**作为** 租户管理员
**我希望** 能够管理考场和分配座位
**以便** 考生知道在哪里考试

### 验收标准

1. **场景1**: 创建考场
   - **当** 租户管理员添加考场
   - **那么** 需要填写考场名称、地址
   - **并且** 可以添加多个教室

2. **场景2**: 创建教室
   - **当** 租户管理员为考场添加教室
   - **那么** 需要填写教室名称、容量
   - **并且** 系统计算总容纳人数

3. **场景3**: 触发座位分配
   - **当** 报名结束且需要分配座位
   - **那么** 管理员触发座位分配
   - **并且** 系统按策略自动分配

4. **场景4**: 座位分配策略
   - **当** 系统执行座位分配
   - **那么** 可以选择：按岗位分组 / 按时间先后 / 随机
   - **并且** 自动填充教室、遵守容量限制

5. **场景5**: 手动调整座位
   - **当** 自动分配完成后
   - **那么** 管理员可以手动调整座位
   - **并且** 调整后更新准考证信息

---

## 用户故事 6: 成绩管理

**作为** 租户管理员
**我希望** 能够管理考生成绩
**以便** 考生可以查询成绩

### 验收标准

1. **场景1**: 导入成绩
   - **当** 考试结束需要录入成绩
   - **那么** 可以上传CSV文件导入成绩
   - **并且** 系统验证数据格式

2. **场景2**: 计算总分
   - **当** 成绩导入完成
   - **那么** 系统自动计算每个考生的总分
   - **并且** 对比及格分数线

3. **场景3**: 考生查看成绩
   - **当** 成绩录入完成
   - **那么** 考生可以登录查看成绩
   - **并且** 看到各科目成绩和总分

4. **场景4**: 录入面试结果
   - **当** 笔试通过后需要面试
   - **那么** 可以录入面试成绩
   - **并且** 标记最终录用状态

## 用户故事 8: 单一账户多租户

**作为** 考生
**我希望** 使用一个账户报名多个单位的考试
**以便** 简化操作

### 验收标准

1. **场景1**: 跨租户报名
   - **当** 考生已有一个账户
   - **那么** 可以报名A公司的考试
   - **并且** 可以报名B公司的考试
   - **并且** 使用同一账户

2. **场景2**: 统一报名列表
   - **当** 考生查看 "我的报名"
   - **那么** 可以看到所有租户的报名记录
   - **并且** 按租户筛选

## 核心业务流程图

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         考生视角的完整流程                                   │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. 注册/登录                                                             │
│       ↓                                                                   │
│  2. 浏览考试 → 选择考试和岗位                                              │
│       ↓                                                                   │
│  3. 填写报名表 → 上传证件                                                 │
│       ↓                                                                   │
│  4. 提交报名                                                             │
│       ↓                                                                   │
│  5. 自动审核结果：                                                        │
│       ├─ 通过 → 进入支付（如果收费）                                       │
│       ├─ 拒绝 → 结束                                                      │
│       └─ 待人工审核 → 进入审核队列                                         │
│       ↓                                                                   │
│  6. 人工审核（一审 → 二审）                                                │
│       ↓                                                                   │
│  7. 审核通过 → 进入支付（如果收费）                                        │
│       ↓                                                                   │
│  8. 支付成功 → 生成准考证                                                 │
│       ↓                                                                   │
│  9. 座位分配 → 更新准考证座位信息                                           │
│       ↓                                                                   │
│  10. 查看/下载准考证 → 前往线下考试                                         │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```
