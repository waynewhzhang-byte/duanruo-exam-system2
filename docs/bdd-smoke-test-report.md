## ❌ 失败的场景（11个）

### 问题分类

#### 1. 认证/授权问题（7个场景）

**问题描述**: 审核员登录失败，返回 422 错误

**影响场景**:
1. 一级审核员审核通过
2. 二级审核员查看待审核任务
3. 二级审核员审核通过
4. 一级审核员审核拒绝
5. 二级审核员审核拒绝
6. 审核员查看审核历史
7. 审核员查看审核统计

**错误信息**:
```
[response 422] http://localhost:3000/api/v1/auth/login
Failed to load resource: the server responded with a status of 422 (Unprocessable Entity)
```

**根本原因**: 
- 审核员账号可能不存在或密码不正确
- 登录 API 返回 422 而不是 401

**建议修复**:
1. 检查测试数据准备脚本，确保审核员账号已创建
2. 验证审核员账号的用户名和密码
3. 检查后端登录 API 的错误处理逻辑

---

#### 2. 前端路由问题（1个场景）

**场景**: 考生查看自己的成绩

**问题描述**: `/my-scores` 路由返回 404

**错误信息**:
```
[response 404] http://localhost:3000/my-scores
Failed to load resource: the server responded with a status of 404 (Not Found)
```

**建议修复**:
1. 检查前端路由配置
2. 确认成绩查询页面是否已实现
3. 验证路由路径是否正确

---

#### 3. 超时问题（3个场景）

**场景**:
1. 一级审核员审核通过 - 审核详情页面
2. 二级审核员查看待审核任务 - 验证任务
3. 二级审核员审核通过 - 审核详情页面

**问题描述**: 页面加载超时（5秒）

**错误信息**:


**建议修复**:

3. 优化 API 响应时间

## 🔍 详细问题分析

### 1. 认证流程问题

**现象**:
- 考生登录成功 ✅
- 租户管理员登录成功 ✅
- 审核员登录失败 ❌

**可能原因**:
1. 审核员账号未在数据库中创建
2. 审核员账号的租户关联不正确
3. 审核员角色权限配置问题

**验证方法**:
```sql
-- 检查审核员账号
SELECT * FROM users WHERE username IN ('bdd_reviewer1', 'bdd_reviewer2');

-- 检查用户角色
SELECT * FROM user_tenant_roles WHERE user_id IN (
  SELECT id FROM users WHERE username IN ('bdd_reviewer1', 'bdd_reviewer2')
);
```

---

### 2. API 端点问题

**401 未授权错误**:
```
[response 401] http://localhost:3000/api/v1/reviews/pending
[response 401] http://localhost:3000/api/v1/reviews/statistics
[response 401] http://localhost:3000/api/v1/applications/my
```

**可能原因**:
1. JWT Token 未正确传递
2. Token 已过期
3. 权限检查失败

---

### 3. 前端路由问题

**404 错误**:
```
[response 404] http://localhost:3000/my-scores
```

**建议**:
1. 检查 `web/app` 目录下是否有 `my-scores` 路由
2. 确认路由命名是否正确
3. 验证路由是否需要认证
