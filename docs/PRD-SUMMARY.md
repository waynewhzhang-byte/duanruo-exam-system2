# PRD 核心功能摘要

> 端若数智考盟 - 多租户在线招聘考试报名管理平台

---

## 1. 产品概述

### 1.1 产品定位

**端若数智考盟**是一个**考试报名管理平台**，为招聘企业提供完整的在线报名、审核、准考证发放解决方案。

### 1.2 核心用户

| 用户角色 | 描述 | 典型场景 |
|----------|------|----------|
| 平台管理员 | 运营方 | 创建租户、监控系统 |
| 租户管理员 | 招聘企业HR | 发布考试、审核报名、管理考场 |
| 审核员 | 企业员工 | 审核考生资料 |
| 考生 | 应聘者 | 报名考试、上传资料、打印准考证 |

## 3. 核心业务流程

### 3.1 租户入驻流程

```
1. 超级管理员登录平台
2. 创建新租户（名称、代码、联系人）
3. 系统自动：
   - 创建 PostgreSQL schema: tenant_{code}
   - 创建 MinIO bucket: tenant-{code}-files
   - 初始化默认表结构
4. 为租户分配管理员账户
5. 租户管理员登录配置
```

### 3.2 考试发布流程

```
1. 租户管理员登录管理后台
2. 创建考试基本信息
3. 配置报名时间窗口
4. 设置是否收费及金额
5. 添加招聘岗位
6. 配置岗位要求（年龄、学历等）
7. 配置报名表单字段
8. 发布考试（状态变为 OPEN）
```

### 3.3 考生报名流程

```
1. 考生访问平台
2. 浏览考试列表
3. 选择报考单位（租户）
4. 选择考试和岗位
5. 填写报名信息
6. 上传证件照片
7. 确认提交
8. 系统执行自动审核
9. 状态更新：
   - 自动通过 → APPROVED
   - 自动拒绝 → AUTO_REJECTED
   - 需要人工 → PENDING_PRIMARY_REVIEW
10. 如需支付 → 跳转支付 → PAID → APPROVED
```

### 3.4 审核工作流

```
【一审】
1. 一审审核员登录
2. 从队列拉取任务
3. 系统锁定任务10分钟
4. 查看报名信息和资料
5. 做出决策：
   - 通过 → PRIMARY_PASSED → 自动进入二审
   - 拒绝 → PRIMARY_REJECTED → 流程结束
   - 退回 → RETURNED → 考生补充材料

【二审】
1. 二审审核员登录
2. 从队列拉取任务
3. 审核同上
4. 做出最终决策：
   - 通过 → APPROVED
   - 拒绝 → SECONDARY_REJECTED
```

### 3.5 准考证发放流程

```
1. 报名状态变为 APPROVED
2. 系统自动生成准考证
3. 考生可查看/下载
4. 考场座位分配后
5. 更新准考证座位信息
6. 考生获取最终准考证
```

## 5. 状态定义

### 5.1 考试状态

| 状态 | 描述 |
|------|------|
| DRAFT | 草稿 |
| REGISTRATION_OPEN | 报名开放 |
| OPEN | 开放 |
| CLOSED | 已关闭 |
| IN_PROGRESS | 进行中 |
| COMPLETED | 已完成 |

### 5.2 报名状态

| 状态 | 描述 |
|------|------|
| DRAFT | 草稿 |
| SUBMITTED | 已提交 |
| AUTO_PASSED | 自动通过 |
| AUTO_REJECTED | 自动拒绝 |
| PENDING_PRIMARY_REVIEW | 待一审 |
| PRIMARY_PASSED | 一审通过 |
| PRIMARY_REJECTED | 一审拒绝 |
| PENDING_SECONDARY_REVIEW | 待二审 |
| APPROVED | 审核通过 |
| SECONDARY_REJECTED | 二审拒绝 |
| PAID | 已支付 |
| TICKET_ISSUED | 准考证已发放 |
