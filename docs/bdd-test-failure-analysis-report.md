- 场景57: 一级审核员审核拒绝 - 点击"审核拒绝"按钮超时
- 场景58: 二级审核员查看待审核任务 - 任务列表加载超时
- 场景59: 二级审核员审核通过 - 查看考生信息超时
- 场景60: 二级审核员审核拒绝 - 点击"审核拒绝"按钮超时
- 场景61: 审核员批量审核 - 点击"批量审核通过"按钮超时

**根本原因**:
虽然我们在阶段2.6修复了5个硬编码的5000ms超时，但**仍有其他步骤定义文件中存在硬编码的5000ms超时**。

**证据**:
```typescript
// exam-management.steps.ts:149
When('我点击"统计分析"标签', async function (this: CustomWorld) {
  // 可能包含硬编码的 timeout: 5000
});

// review.steps.ts:30
When('我在审核详情页面', async function (this: CustomWorld) {
  // 可能包含硬编码的 timeout: 5000
});
```

**修复方案**:
搜索所有步骤定义文件中的硬编码超时，将5000ms改为20000ms。

2. **选择器不匹配** - 步骤定义中的选择器与实际页面元素不匹配

**修复方案**:
1. 检查前端页面是否正确显示错误提示
2. 更新步骤定义中的选择器
3. 使用更灵活的选择器策略（多个备选选择器）


- `score-management.steps.ts`: 保持 "我确认删除"
- `seat-tenant.steps.ts`: 改为 "我确认删除租户"

```typescript
    `text=${errorMessage}`,
    `[role="alert"]:has-text("${errorMessage}")`,
    `.error:has-text("${errorMessage}")`
  ];
  
  let found = false;
  for (const selector of errorSelectors) {
    const element = this.page.locator(selector);
    if (await element.count() > 0) {
      await expect(element).toBeVisible({ timeout: 20000 });
```
