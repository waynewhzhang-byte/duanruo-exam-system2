# 多租户考试报名系统 - 基于角色的UI功能映射

## 文档概述

本文档定义了系统中所有角色及其对应的前端UI页面和功能需求，基于SAAS-PRD.md需求和后端API分析。

---

## 1. 角色定义总览

### 1.1 全局角色
- **SUPER_ADMIN** (超级管理员) - 平台级管理员
- **CANDIDATE** (考生) - 全局用户，可报名多个租户的考试

### 1.2 租户级角色
- **TENANT_ADMIN** (租户管理员) - 管理特定租户的所有事务
- **EXAM_ADMIN** (考试管理员) - 管理被分配的特定考试
- **PRIMARY_REVIEWER** (一级审核员) - 初级人工审核
- **SECONDARY_REVIEWER** (二级审核员) - 复审确认
- **EXAMINER** (考官) - 验证准考证

---

## 3. 缺失功能优先级分类

### P0 (高优先级 - 核心业务功能)
1. **自动审核规则配置** - 影响审核流程
2. **审核员管理** - 核心业务角色管理
3. **准考证模板配置** - 核心业务功能
4. **CSV批量导入成绩** - 大规模考试必需
5. **报名表单条件逻辑** - 复杂表单必需

### P1 (中优先级 - 重要功能)
1. **通知管理系统** - 用户体验重要
2. **考官准考证验证** - 考试现场必需
3. **科目管理完善** - 多科目考试必需
4. **座位分配可视化** - 大规模考场管理
5. **成绩统计分析** - 数据分析需求
